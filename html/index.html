Load Balancer Simulation

This program simulates a load balancer managing multiple web servers to handle incoming requests efficiently. The load balancer distributes requests to the available servers, tracks their status, and logs the simulation events to a file for analysis.

Setup and Execution

To compile and run the simulation:

  • Ensure you have a C++ compiler installed.
  • Run the Makefile with make.
  • Run the executable with ./main.

Algorithm

The simulation implements a dynamic load balancing algorithm. It continuously allocates and deallocates servers based on the current load and generates new requests randomly during execution.

Load Balancer Design

The load balancer in this project is designed to distribute requests efficiently across multiple servers while ensuring optimal resource utilization and request handling.

  • Load Balancing Algorithm: The load balancer employs a Round Robin algorithm that allows servers to process incoming requests dynamically. Each server takes turns in a cyclic manner to handle requests from a centralized queue, ensuring an equitable distribution of load.
  • Queue Implementation: Requests are stored in a First In, First Out (FIFO) queue. This design guarantees that requests are processed in the order they arrive, minimizing wait times for earlier requests.
  • Dynamic Server Allocation: The system can dynamically allocate or deallocate servers based on the current load. If the request queue grows beyond a specified limit, additional servers are activated. When the load decreases, excess servers are shut down to conserve resources.
  • Request Handling: Each server processes requests independently. If a server completes a request, it immediately checks the queue for any pending requests to ensure continuous operation.
  • IP Range Blocking: The load balancer features a firewall mechanism that blocks requests from specific IP ranges. This acts as a defense mechanism against unauthorized access or Denial of Service (DOS) attacks by rejecting requests outside a predefined IP range.
  • Performance Metrics: Upon completion of the simulation, the system outputs performance metrics such as the number of requests processed, average processing time, and server utilization rates. These metrics are critical for evaluating the efficiency of the load balancer.

Execution Flow

  1. Initialize the log manager and load balancer.
  2. Accept user input for the number of servers and simulation duration.
  3. Generate initial requests based on the number of servers.
  4. Process requests in a loop until the specified run time is reached.
  5. Log the final status, including total servers, requests processed, and server activity.

Conclusion

This load balancer simulation serves as a useful tool for understanding load distribution among servers and can be extended or modified for more complex scenarios.
